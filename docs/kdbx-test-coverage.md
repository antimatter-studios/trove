# kdbx test coverage matrix (historical — vendored 0.7.33)

> **As of v0.0.10.0 this document is historical.** The vendored `keepass` 0.7.33 fork has been retired in favour of the published `keepass = "0.12.5"` crate, and the test suite has been ported and moved to [crates/keepass-spec-tests/tests/](../crates/keepass-spec-tests/tests/). Some of the gaps listed below — most notably the two truncation parser panics — are expected to be fixed in 0.12.5; the ported tests will surface the actual situation. Keeping this doc as a record of what 0.7.33 looked like at the moment of migration.

Generated by reading `vendor/keepass/tests/{spec_round_trip,broken_files,keyfile_formats,binary_pool,cross_tool}.rs` (the now-deleted vendor tree).
This is the gap inventory for the upcoming 0.12.x migration. Cells:

- **pass**: a `#[test]` exists, runs by default, and asserts the behaviour.
- **ignored**: a `#[test]` exists but is `#[ignore]`'d with a precise rationale (so it shows up under `cargo test -- --ignored`). These are gaps, not failures.
- **n/a**: feature is not exposed by 0.7.33's public API.

## Migration triggers

The following are the gates for un-ignoring tests / closing gaps. Each is a candidate PR for the 0.12.x migration:

1. **Fix `TruncatedAtHeader` panic** (`format/kdbx4/parse.rs:165`). Bound-check the slice access. Likely already fixed upstream — verify against master before re-implementing.
2. **Fix `TruncatedAtPayload` panic** (`hmac_block_stream.rs:22`). Same shape — slice access without length check.
3. **Add an XML-injection test harness** (private API or `#[cfg(test)] pub`) so we can express `NoRootGroup`, `MultipleRootGroups`, `EmptyXml`, `MalformedUtf8`, `DanglingBinaryRef`, `MissingEntryUuid`, `MissingGroupUuid`, `BadXmlInnerHeader` directly. Once present, the eight `#[ignore]`d XML mutations can be unignored.
4. **Add a `vendor/keepass/fuzz/` harness** with `cargo-fuzz` targets `fuzz_parse_kdbx4`, `fuzz_variant_dictionary`, `fuzz_inner_header`. This catches the same class of slice-OOB bugs as #1 and #2.
5. **KDBX 3 round-trip matrix.** 0.7.33's writer is KDBX4 only; un-ignore once 0.12.x's KDBX3 writer is in.
6. **Property-based round-trip** (`proptest`). Generate arbitrary `Database` shapes, save, reopen, assert equal.
